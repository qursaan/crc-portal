1        Introduction

The purpose of testbed portal is to provide a user web access to both the testbed infrastructure and/or simulation server to execute experiments remotely. The target solution should provide an interface for users to access testbeds. The portal function allows browsing, access and reserving resources.

Portal application is integrate with scheduling system, disk Imaging, OMF Manager, and simulation server. Portal interface is enhancement and simplify the user functions including registration, reservation, and experimentation test.

Figure 1 show the user interface of portal dashboard. This screen is show once the user logged into the system.

5        Lab Panel

5.1      Request a slice

The main interest of user function is to reserve the testbeds/simulation node to able to running the purpose test. In this step, the user has to create new request and reserve a desired node for set of time. To request new slice from dashboard screen click on “Request Slice” under “Lab Panel” section (or click here)

Set Reservation Information

Here, the user has to specify necessary information such as.

1.      Slice Name

The user can write any name for marking his own slice in history.

2.      Specify the Servers/Machine Type

The user can choose between two types of servers, Simulation Server and Testbed Nodes (Figure 9).

3.      Request Plan

The user can choose between Normal/Scheduling request or Urgent request (Figure 10)

·         Normal Request: In this plan, the user has to request slice before actual reservation in one-day advance to choose from all available resources.

·         Urgent Request: In this plan, the user has to request slice in the current day (Figure 11) and with only available resources.

4.      Slice Time

The user has to specify the reservation slice time need to running the test. The slice time should be chosen from current free timetable (Figure 12).

5.      Determine the Hardware Specifications

In this step, the user determines the hardware specification of the servers and/or nodes need to be included in experiment

·         (Figure 13) show a resources available in case of using Testbed Nodes (in Figure 9)

o   In case of choosing “USRP” Resource, you will see a list of all available frequencies range. User should choose at least one frequency range in this case (Figure 14)

·         (Figure 15) show all virtual machine available in case of using Simulation Server (in Figure 9)

6.      Operating System Image

In this step, the user has to choose the image of the server in case choose a simulation server or a virtual machine. There are some default images will be available to all users. However, the user can set/upload an image from the Cloud disk

·         (Figure 16) show a images available in case of using Testbed Nodes (in Figure 9)

·         While (Figure 17) show image in case of using Simulation Server (in Figure 9)

6        Control and Manage Nodes

Once the user access the control panel based on his slice and scheduling “Current Requests” Page (see Figure 22), the “Control panel” page is opened (see Figure 24). The following are the main parts and functions on this page are:

1.      Instructions tabs: list a basic running instructions

2.      Load Images tab: to load images on selected nodes

3.      Script Manager tab: to run and execute custom scripts

4.      Console tab: contain tools to remote access node via controller or simulation server using SSH.

5.      Save Images:  to save current node image on user local space for future use.

6.      Remote controls tabs: to remote start/shutdown or restart the node.

7.      Timer window: to view remaining slice time

8.      Activity Log Panel: log the recent user operation and status.

 

Figure 24: Control Panel UI (1)

 

6.1      Load Images

After preparing nodes and/or virtual machines, the user should load the proper image or default image in order to starting experiment.

To load image on target node(s) do the following steps:

1.      Press on the “Load Image” tab,

2.      Choose the target nodes from node list

3.      Select the base image from the list of images

4.      Press “Load Image” button.

The progress of loading image will appeared on “Activity Log” panel (see Figure 25).

 

Figure 25: Control Panel UI (2)
